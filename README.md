# Тестовое задание на позицию Outstaff Frontend-разработчика

Нужно разработать интерфейс приложения для просмотра и управления статьями, состоящий из двух страниц.

## Продуктовые требования

### Главная страница

- Показывает список статей (5–10 статей) в виде карточек, отсортированных по заголовку.
- Каждая статья содержит:
  - заголовок
  - краткое описание (первые 100 символов контента)
  - кнопку "Просмотреть" для перехода на страницу статьи
  - кнопку "Удалить" для удаления статьи из списка
- Список статей должен загружаться автоматически при входе на страницу
- После удаления статьи список обновляется без перезагрузки страницы

### Страница статьи

- Должна содержать:
  - заголовок статьи
  - полный текст статьи
  - дату создания (можно задать в моковых данных)
  - кнопку "Назад" для возврата к списку статей
- Данные статьи подгружаются сразу при входе на страницу

## Технические требования

- Приложение разработано с использованием TypeScript, React и [TanStack Query](https://tanstack.com/query)
- Использован моковый REST API, реализованный через [MSW](https://mswjs.io/) с эндпоинтами:
  - GET /articles — список статей (id, title, content)
  - GET /articles/:id — детали статьи
  - DELETE /articles/:id — удаление статьи
- Клиент для API сгенерирован через [openapi-ts](http://openapi-ts.dev)
- Роутинг выполнен с использованием [TanStack Router](https://tanstack.com/router) с роутами:
  - / (список статей)
  - /articles/:id (страница статьи)
- Фреймворк UI или стили на ваше усмотрение (например, [Tailwind](https://tailwindcss.com/), CSS modules или inline). Главное — минималистичный, но аккуратный дизайн
- При переходах по ссылкам страница не перезагружается
- MSW активен только в development-режиме (условная инициализация)
- Проект грамотно декомпозирован на пакеты в рамках nx-репозитория (например, отдельные пакеты для UI-компонентов, API-клиента, бизнес-логики и т.д.) [[документация nx](https://nx.dev/)]
- Исходный код решения должен быть выложен с вашего аккаунта на [GitHub](http://github.com/) с readme файлом, содеражщий описание проекта и инструкции для его запуска.

## Опциональные

- Пакетный менеджер [pnpm](https://pnpm.io/)
- Загрузка данных о статьи загружается через route api
